## Core Positioning

李伟民是一名面向 B 端硬件、AI 终端与海外业务的高级产品经理。职业主线从音视频会议与远程医疗产品，逐步发展到 AI 桌面算力产品线、MegaCube 海外业务、白标/OEM 定制和本地 AI Agent 工作流。

## Product Detail Pages

个人网站需要承载过去产品作品的详情，而不是只展示简历摘要。过去通过二维码链接整理过的作品，可以作为每个项目详情页的外部资料入口。

推荐每个项目详情页使用统一结构：

- 项目背景：行业、客户、场景、当时的问题
- 我的角色：负责范围、决策权限、协作对象
- 用户与业务问题：关键角色、核心流程、真实痛点
- 产品方案：信息架构、核心功能、软硬件组合、交付方式
- 项目结果：落地客户、出货量、项目金额、效率或体验提升
- 作品资料：二维码链接、原型链接、演示视频、方案文档、截图
- 复盘：当时做对了什么，今天看还有哪些限制
- AI / Agent 二次重造：如果用当前 AI 和 Agent 能力重做，会如何改造产品和业务链路

## AI / Agent Rebuild Column

这是个人网站的关键新方向。它不只是展示“过去做过什么”，而是展示“基于现在的 AI 能力，我如何重新理解和重造这些业务”。

建议新增一个一级栏目：

> AI Rebuild

中文可以叫：

> AI 二次重造

栏目定位：

- 把历史项目当作真实业务样本
- 用 Agent、端侧 AI、本地大模型、多模态识别、自动化工作流重新设计
- 展示产品经理对新技术落地路径的判断，而不是只展示工具使用能力

适合重造的方向：

- 远程医疗：从视频协同平台升级为手术/超声/阅片场景的 AI 辅助协同系统
- 金融双录：从录制与质检工具升级为实时合规 Agent、质检 Agent、风险提示 Agent
- 会议产品：从会议功能集合升级为会前准备、会中协同、会后纪要与任务追踪的智能工作台
- MegaCube：从 AI 算力硬件升级为本地 Agent 运行底座和行业解决方案入口
- 海外渠道：从销售资料分发升级为代表处市场情报、报价、竞品分析和销售话术 Agent

每个 AI 重造专题建议回答：

1. 原业务为什么成立
2. 原产品在当时解决了什么问题
3. 原方案的边界在哪里
4. 当前 AI / Agent 能力带来了什么新变量
5. 新产品形态是什么
6. 商业模式或交付方式如何变化
7. MVP 应该怎么做
8. 需要哪些数据、硬件、模型和系统集成

## External Portfolio Links

过去的二维码作品链接应统一整理成一个资料清单，然后在网站中按项目挂载。

资料类型建议包括：

- 原型链接
- 产品文档
- 演示视频
- 项目截图
- 客户案例
- 培训材料
- 二维码图片

不要只把二维码图片放在页面上。更推荐：

- 页面上展示项目摘要、关键截图和结果
- 提供“查看原型 / 查看作品资料”的链接按钮
- 二维码作为移动端扫码入口或补充入口
